Directives are special attributes with the "@" prefix that Quantum JS has built in out of the box. They are used to make you development alot faster and apply changes to the DOM when your View Model updates.

The @If directive is used when you are wanting to render an element dependent on a truthy value in your view model.
For example, lets say you have a div which should be displayed when a user doesn't fill in an input field. We would use the @If.
As you can see the property showErr is true, which means the element with the @If directive will be displayed.


The @Bind directive is very useful. You can use this directive to bind a property in your View Model to an attribute on an element. For instance, if you want to apply a class to an element if an error occurs. You would use the @Bind:class attribute to do so.

Binding Type Description
@Bind:id String Allows you to bind your prop to the ID of the element
@Bind:class String Allows you to bind your prop to the CLASS of the element
@Bind:disabled Boolean Will add a disbaled attribute to the element depending on the value of your prop
@Bind:checked Boolean This will set a checked attribute to the element depending on the value of your prop
@Bind:href String Will set the href attribute to the value of your prop
The @On directive is used to bind a users action to an element. For example, if you have a button to alert a user's name once it's clicked. You would use the @On:click directive and this would add a click event listener to that element.
Binding Type Description
@On:click String This will add a click event to your element
@On:enter String This enables you to add an enter event to your element
@On:submit String This allows you to add a submit event to a form
@On:change String This allows you to add an onchange event to your element
@On:enter String This allows you to add an enter key event to your element
@On:scroll String This allows you to add a scroll event to your element
The @Reflect directive is a very special attribute. By adding this attribute to an element, you bind the value of that element to the value of your View Model, property, and the propery inside your View Model is bound to the value of the element. This is called Two way data binding. Any updates that happen to the controller and model, will update the DOM, as well as any updates that happen in the DOM will update your View Model
In the example below, we have an input which is bound to userInput in the view model. And a heading which is also bound to the same property.
