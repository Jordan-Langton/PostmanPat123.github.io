The View Model is a central place for the options and propterties of your View Controller. These options and properties are vital for your view to work properly. Lets see what we mean by options and properties.
Your options are there to help your controller understand more about where things are in the current view. For example your View Controller needs to know where it needs to be rendering the current view too. Lets take a look at what options you can have.
The ele property specifies where the current view will be rendered to. For example, lets say we have our main root div, and we would like to render our view into it. All we do is pass the ID of that element to the ele prop.
The style property specifies where the style sheet you would like to use for your current view is in your view folder.
The view property specifies where the html file you would like to use for your current view is in your view folder.
The folderName property specifies the name of the folder which contains the current views files.
The components property is a list of all the components you are going to be using inside of your view template.
Your state is all the properties your controller will ever need or use. For example, say you need to save some user data. You would setup a property in your state, we will call it userData, and whenever you would want to save that user data, you would use the setState() method and update you state property.
As you can see the State Object isn't just an empty object for you to change. It has two properties. The first being scoped and the second being global These properties both have special use cases, for you to utilize. Lets take a look at what the do.
The scoped object property would be used if you are wanting to set a property for your View Controller that will only be used on the current view. Meaning it scopes that prop to this view only and other views are not able to access that property. For example, lets say you are building a view which displays an error if you dont't fill in a field. this error will show if you set showError to true. This showError property would go inside the scoped object.
The global object property would be used if you are wanting to set a property for your View Controller that will not only be accessed by the current View Controller but others as well. How though? Well if you put a property inside the global object, it will not only be set inside the controller, but the applications global state object as well. Meaning that property is accessable via state on any other view.
The scoped/global object property is the same as the global object, but the only difference is that the scoped object inside of the global object will be set under the View Controller's name in the application's global state object.
